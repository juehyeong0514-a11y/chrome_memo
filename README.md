# Web Annotation Floating Pen

일반 웹페이지 위에 필기할 수 있는 Chrome Manifest V3 확장 프로그램입니다. 웹페이지 오른쪽에 작은 원형 버튼을 띄우고, 버튼을 누르면 펜/형광펜/지우개 도구막대가 열립니다.

## 주요 기능

- Chrome Manifest V3 기반 확장 프로그램
- Chrome 확장 프로그램 아이콘 클릭 시 작은 팝업 표시
- 팝업에서 전체 ON/OFF 전환
- 현재 페이지에서 확장 사용 가능 여부 표시
- 팝업에서 도구 위치 초기화
- 웹페이지 위에 떠 있는 원형 필기 버튼
- 원형 버튼 클릭으로 도구막대 열기/닫기
- 상단 드래그 손잡이 또는 원형 버튼으로 도구막대 이동
- 도구 버튼, 색상칩, 굵기 버튼 클릭 시 드래그 오작동 방지
- 화면 밖으로 나가지 않도록 8px 여백 기준 위치 보정
- 창 크기 변경, 전체화면 전환, 브라우저 확대/축소, DevTools로 인한 화면 크기 변경 대응
- `chrome.storage.local`에 도구막대 위치 저장
- URL별 필기 데이터 저장 및 새로고침 후 복원
- 문서 좌표 기반 필기 저장으로 스크롤 후에도 위치 유지
- Shadow DOM 기반 도구막대 UI로 웹페이지 CSS 충돌 방지

## 도구막대

도구막대는 작고 깔끔한 패널 형태입니다.

- 펜
- 형광펜
- 지우개
- 실행 취소
- 다시 실행
- 필기 숨기기/보이기
- 필기 모드/탐색 모드 전환
- 전체 지우기
- 원형 색상칩
- 얇게/보통/굵게 굵기 선택

선택된 도구, 색상, 굵기는 각각 하나만 강조 표시됩니다.

## 설치 방법

1. Chrome 주소창에 `chrome://extensions`를 입력합니다.
2. 오른쪽 위 **개발자 모드**를 켭니다.
3. **압축해제된 확장 프로그램을 로드합니다**를 클릭합니다.
4. 아래 폴더를 선택합니다.

```text
c:\Users\User\Desktop\chorm_meno
```

## 사용 방법

### 팝업

1. Chrome 툴바의 확장 프로그램 아이콘을 클릭합니다.
2. ON/OFF 스위치로 확장을 켜거나 끕니다.
3. 현재 페이지에서 사용할 수 없는 경우 팝업에 안내 문구가 표시됩니다.
4. **도구 위치 초기화** 버튼을 누르면 도구막대가 화면 오른쪽 가운데로 이동합니다.

OFF 상태에서는 도구막대와 캔버스가 숨겨지고 필기 입력이 비활성화됩니다. 저장된 필기 데이터는 삭제되지 않으며, 다시 ON으로 전환하면 기존 필기가 다시 표시됩니다.

### 도구막대 이동

- 접힌 상태에서는 원형 버튼을 드래그해 이동할 수 있습니다.
- 펼친 상태에서는 패널 상단의 드래그 손잡이를 잡고 이동할 수 있습니다.
- 도구막대 위치는 화면 크기 대비 비율로 저장됩니다.

```js
{
  toolbarPosition: {
    xRatio: 0.9,
    yRatio: 0.5
  }
}
```

### 필기

- `✏ 필기 모드`에서는 웹페이지 위에 필기할 수 있습니다.
- `🖱 탐색 모드`에서는 필기는 보이지만 클릭이 웹페이지로 전달됩니다.
- 스크롤해도 필기는 원래 웹페이지 내용 위치에 붙어 있습니다.
- 전체 지우기는 확인 창 이후 현재 페이지의 필기를 모두 삭제합니다.

## 단축키

- `Esc`: 탐색 모드로 전환
- `Ctrl+Z`: 실행 취소
- `Ctrl+Shift+Z`: 다시 실행
- `Ctrl+Y`: 다시 실행

입력창, textarea, select, contenteditable 요소에 커서가 있을 때는 웹페이지 기본 단축키를 방해하지 않습니다.

## 저장 방식

현재 필기 데이터는 탭 메모리에서만 관리됩니다. 탭이나 브라우저를 닫거나 새로고침하면 필기는 삭제됩니다.

`chrome.storage.local`에는 펜 설정과 도구막대 위치만 저장합니다.

필기 중에는 내부적으로 아래와 같은 벡터 데이터를 사용합니다.

```json
{
  "tool": "pen",
  "color": "#ff0000",
  "width": 4,
  "opacity": 1,
  "points": [
    { "x": 100, "y": 500 },
    { "x": 104, "y": 505 }
  ]
}
```

URL 키는 페이지 전환 감지에 사용합니다. hash 변경을 같은 페이지로 볼지 여부는 [src/utils.js](src/utils.js)의 `includeHashInPageKey` 값으로 조정할 수 있습니다.

## 현재 제한사항

- 첫 버전은 document 전체 스크롤을 우선 지원합니다.
- 내부 스크롤 컨테이너별 필기는 아직 지원하지 않습니다.
- iframe 내부 필기는 페이지 구조와 브라우저 보안 정책에 따라 제한될 수 있습니다.
- Chrome 내장 PDF 뷰어에서는 정상 동작하지 않을 수 있습니다.
- Chrome 내부 페이지, Chrome Web Store, 확장 프로그램 관리 페이지에서는 content script가 차단됩니다.
- 이런 페이지에서는 팝업에 “이 페이지에서는 확장 프로그램을 사용할 수 없습니다.” 안내가 표시됩니다.
